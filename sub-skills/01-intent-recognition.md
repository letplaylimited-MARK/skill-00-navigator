# Sub-Skill 01 · 意图识别引擎

**所属 Skill**：Skill 00 · Navigator  
**版本**：v1.0.0  
**职责**：将用户自然语言输入解析为结构化意图，计算路由置信度

---

## 意图类型注册表

| 意图类型 ID | 中文描述 | 触发关键词 | 首选路由 |
|------------|---------|-----------|---------|
| `NEW_PROJECT_IDEA` | 新项目想法/模糊需求 | 我想做/我有个想法/如何构建/能不能做 | Skill 01 |
| `REQUIREMENTS_EXIST` | 已有需求文档待加工 | 需求文档/PRD/已经整理了/这是需求 | Skill 02 |
| `OPEN_SOURCE_SCOUT` | 寻找开源工具/库 | 开源/有什么库/推荐工具/哪个框架更好 | Skill 03 |
| `PROJECT_PLANNING` | 项目执行计划 | 怎么执行/实施计划/分几个阶段/排期 | Skill 04 |
| `QUALITY_CHECK` | 测试/验收/质量保证 | 测试/验收/检查/发布前/质量 | Skill 05 |
| `FULL_PIPELINE` | 从零到一完整流程 | 完整项目/端到端/从头开始/完整的AI项目 | 01→03→02→04→05 |
| `AMBIGUOUS` | 无法确定 | （无明确关键词匹配） | 追问 |

## 置信度计算规则

### 三档路由逻辑

```
置信度 ≥ 80%  →  直接路由，输出交接包，说明路由理由
置信度 60-79% →  输出"我倾向于将您路由到 Skill XX，请确认？"
置信度 < 60%  →  启动追问流程（最多3次，不超过3轮）
```

### 置信度评分维度

| 维度 | 权重 | 满分条件 |
|------|------|---------|
| 关键词匹配度 | 40% | ≥2个强触发词命中同一意图 |
| 上下文连贯性 | 30% | 前后消息意图一致 |
| 意图唯一性 | 20% | 只匹配一个意图类型（无歧义） |
| 用户明确度 | 10% | 用户直接说明想要什么 |

### 计算示例

```
输入："我想用Python做一个分析客户评论情绪的工具"

关键词匹配：
  - "做"+"工具" → NEW_PROJECT_IDEA (命中)
  - "分析"+"情绪" → 无直接匹配
  - Python 技术栈 → 增强 NEW_PROJECT_IDEA

置信度：
  - 关键词匹配度：35/40（命中2个强词，技术栈清晰）
  - 上下文：无上文，默认满分 30/30
  - 意图唯一性：18/20（无歧义）
  - 用户明确度：8/10（目的明确但功能范围待确认）

总分：91% → 直接路由到 Skill 01
```

```yaml
    - "Python"

  route_decision: "DIRECT"   # DIRECT / CONFIRM / CLARIFY

```

## 追问流程规范

当置信度 <60% 时，按以下模板追问：

**第1次追问**（定性）：
```
我想确认您的主要需求方向：
A) 我有个项目想法，需要梳理需求
B) 我有需求文档，需要转化为开发SOP
C) 我需要寻找合适的开源工具/库
D) 其他（请描述）
```

**第2次追问**（定量，仅在第1次仍不清楚时）：
```
您目前项目处于哪个阶段？
A) 只有想法，还没开始
B) 已有需求，准备开始开发
C) 正在开发中，需要解决某个问题
D) 即将发布，需要测试验收
```

**第3次追问**（最后一次）：
```
请用一句话描述您最想让AI帮您完成的具体任务，我将为您精准路由。
```

**3次后仍无法判断**：
```
根据您的描述，我建议从 Skill 01（需求翻译官）开始，它适合处理各类早期阶段的AI项目。
[输出默认路由交接包]
```

---

## 禁止事项

- **禁止** 将意图"猜成"用户可能想要的（应追问确认）
- **禁止** 一次追问超过4个选项（认知负荷过重）
- **禁止** 跳过置信度计算直接路由
- **禁止** 在意图识别阶段执行任何任务
