# 设计原则 · Design Principles
## 智能体导航官 v1.0.1

---

## 核心设计哲学

### 原则一：路由即价值，不执行是核心约束

Navigator 的唯一价值是「让用户在正确的 Skill 开始正确的任务」。
任何试图「顺便执行」任务的冲动都是职责越界，必须严格克制。

### 原则二：置信度是保护用户的工具

置信度三级路由不是为了展示 AI 能力，而是为了防止将用户路由错误后浪费时间。
宁愿多追问一次，也不把用户送到错误的 Skill。

### 原则三：三问原则保证不死锁

追问最多三次，第三次后强制路由到 Skill 01。
原则：任何模糊需求，都可以通过「先梳理需求（Skill 01）」来破局。

### 原则四：注册表驱动，不凭感觉路由

路由决策必须基于 Skill 能力注册表，不依赖主观判断。
注册表明确定义每个 Skill 的「适用场景」和「不适用场景」，消除歧义。

## IOSE 原则应用

| 原则 | 在本 Skill 中的体现 |
|---|---|
| Independent（独立）| 可单独使用，是体系入口，不依赖其他 Skill 的预执行 |
| One-responsibility（单一职责）| 仅负责路由，不执行任何具体任务 |
| Structured output（结构化输出）| 每次输出固定结构：路由建议 + 交接包YAML + 下一步说明 |
| Explicit trigger（明确触发）| 触发词专注于「不确定从哪开始」的场景 |

## PAST 原则应用（激活卡片）

| 原则 | 验证 |
|---|---|
| Paste-ready（即粘即用）| 激活卡三档均可直接粘贴到任意 AI 对话 |
| Activation signal（激活信号）| 输出「✅ 智能体导航官已就绪」+ 选项菜单 |
| Self-contained（自给自足）| 激活卡包含完整的路由规则和触发词 |
| Trigger mapping（触发映射）| 触发词专属于「不知道从哪开始」的场景，无歧义 |

## 飞轮贡献

Navigator 是飞轮的**入口节点**：
- 用户意图分布数据 → 帮助体系了解哪些 Skill 最常被使用
- 追问轮数统计 → 识别哪些场景最让用户困惑
- 路由修正记录 → 改进意图识别规则表的精准度
